This means the sole machine-verification artifact in the evidence chain is out of date. For C4 criticality, a stale machine validation artifact is an evidence quality issue — the "proof" of schema validation does not prove the current file's validity, only a prior version's.

The documentation correctly describes the behavior; the citation quality is adequate for a MEDIUM-criticality document but at C4, a primary source citation (official Claude Code documentation or release notes confirming the plugin namespace behavior) would be stronger.

**Why 0.85 rather than 0.88:**

The stale schema validation JSON is a new finding (not present in prior scoring because it applied to the then-current file). At C4 criticality, a machine-verification artifact that does not match the current file state actively reduces evidence quality. The priority-1 gap closure raises evidence quality significantly from 0.78; the stale artifact partially offsets this gain. The net result — 0.85 — reflects: resolved four-strategy gap (+0.07) partially offset by stale artifact (-0.02) and persistent IN-001 behavioral gap.

**Improvement Path:**
